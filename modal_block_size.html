📦 ขีดจำกัดพื้นที่ 4000 vB

พื้นที่ส่วนนี้คืออะไร?

นี่คือ "ขนาดความจุของบล็อก (Block Size Limit)" กฎกติกาของบิตคอยน์กำหนดขีดจำกัดขนาดข้อมูลเอาไว้ เพื่อป้องกันไม่ให้เครือข่ายบวมเร็วเกินไป และเพื่อให้คอมพิวเตอร์ทั่วไปยังสามารถดาวน์โหลดและเป็น Full Node ได้ พื้นที่ตรงนี้จึงเปรียบเสมือน "อสังหาริมทรัพย์ที่แพงที่สุดในโลกดิจิทัล"

ทำไมในเว็บถึงจำกัดไว้ที่ 4000 vB?

ในหน้าเว็บ Simulator นี้ เราจำลองย่อส่วน (Scale Down) ขนาดความจุลงมาเหลือเพียง 4,000 vB เพื่อให้คุณสามารถเห็นภาพธุรกรรมประมาณ 10-20 รายการเต็มบล็อกพอดี หากเราใช้ขนาดจริง คุณอาจจะต้องนั่งคลิกเลือกธุรกรรมทีละรายการถึง 3,000 ครั้งกว่าบล็อกจะเต็ม ซึ่งคงไม่เหมาะกับการทำ UI ให้ผู้คนเรียนรู้ครับ

ความจุในโลกความเป็นจริง (Mainnet)

  • หลังจากอัปเดต SegWit บล็อกของบิตคอยน์ในโลกจริงถูกจำกัดไว้ที่ 4,000,000 Weight Units (WU)
  • เมื่อแปลงเป็นหน่วย Virtual Bytes (vB) จะตกอยู่ที่ประมาณ 1,000,000 vB (หรือขนาดไฟล์ดิบราวๆ ไม่เกิน 4 MB)
  • ด้วยพื้นที่ 1 ล้าน vB นี้ ในโลกจริง 1 บล็อกจะสามารถบรรจุธุรกรรมโอนเงินมาตรฐานได้ประมาณ 2,000 - 4,000 รายการ ต่อหนึ่งบล็อก (ทุก 10 นาที)

ความคุ้มค่าดูจากอะไร และส่งผลอะไรต่อค่าธรรมเนียม?

เพราะพื้นที่บล็อกมีจำกัด "ผู้ส่ง" จึงต้องประมูลแย่งพื้นที่กันด้วยการแนบค่าธรรมเนียม (Fee) ยิ่งผู้ส่งให้เงินรวมเยอะ และธุรกรรมมีขนาด Byte เล็ก ความคุ้มค่าที่เรียกว่า sat/vB (ซาโตชิต่อไบต์) ก็จะยิ่งสูง

นักขุดจะเรียงลำดับกวาดธุรกรรมที่มีค่า sat/vB สูงสุดใส่กล่องให้เต็ม 1,000,000 vB ก่อนเสมอ ใครที่จ่ายค่าธรรมเนียมด้วยเรท sat/vB ต่ำเกินไป ก็จะถูกทิ้งค้างไว้ใน Mempool ต่อไปเรื่อยๆ จนกว่าคิวจะว่างครับ
