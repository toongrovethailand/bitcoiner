💰 เคลมรายได้ (Block Subsidy)

Subsidy คืออะไร?

Block Subsidy (เงินอุดหนุนบล็อก) คือจำนวนเหรียญบิตคอยน์ที่ "ถูกสร้างขึ้นมาใหม่" เพื่อแจกจ่ายเป็นรางวัลให้กับนักขุดที่ประมวลผลบล็อกได้สำเร็จ นี่เป็นกลไก เดียว ในการผลิตเหรียญเข้าสู่ระบบเศรษฐกิจของบิตคอยน์

สมการการเกิดเหรียญใน Source Code (validation.cpp)
CAmount GetBlockSubsidy(int nHeight, const Consensus::Params& consensusParams) {
    int halvings = nHeight / consensusParams.nSubsidyHalvingInterval;
    if (halvings >= 64) return 0;
    CAmount nSubsidy = 50 * COIN;
    nSubsidy >>= halvings;
    return nSubsidy;
}
📌 อธิบายการทำงานของโค้ด:
  • int halvings = nHeight / consensusParams.nSubsidyHalvingInterval;
    สมการนี้หมายถึง: จำนวนครั้งที่เกิดฮาฟวิ่ง (Halvings) = ความสูงของบล็อกปัจจุบัน / 210,000 (ซึ่ง 210,000 คือจำนวนบล็อกมาตรฐานใน 1 รอบฮาฟวิ่ง) โปรแกรมจะนำมาหารกันเพื่อหาว่าตอนนี้โลกเราอยู่ในยุคที่เท่าไหร่
  • nSubsidy >>= halvings;
    นี่คือตัวดำเนินการคณิตศาสตร์ที่เรียกว่า Bitwise Shift Right ซึ่งมีค่าเท่ากับการเอารางวัลตั้งต้น (50 BTC) มา "หาร 2" ซ้ำๆ ตามจำนวนรอบฮาฟวิ่งที่คำนวณได้ด้านบนนั่นเอง!

ตารางการลดรางวัล (Halving Schedule) 0 - 33

Halving 📦 ช่วงบล็อก (Range) ปีโดยประมาณ รางวัลต่อบล็อก (Sat) ผลรวมในรอบนี้ (Sat)

เหรียญจะมีเกิน 21 ล้าน BTC ได้หรือไม่?

บิตคอยน์เป็นเพียงแค่ซอฟต์แวร์ (Open Source Code) ซึ่งในทางทฤษฎีแล้ว การแก้ไขโค้ดเพื่อพิมพ์เหรียญเพิ่มนั้นสามารถทำได้ง่ายมาก แค่เปลี่ยนตัวเลขในบรรทัดไม่กี่บรรทัด

แต่... การแก้โค้ดไม่ได้หมายความว่าเครือข่ายจะยอมรับ!

กลไกการปกป้องโดยผู้ใช้งาน (User-Actuated Protection)

หากมีนักขุดกลุ่มใหญ่ หรือผู้พัฒนาระบบพยายามอัปเดตโค้ดเพื่อเสกเหรียญให้ทะลุ 21 ล้าน พวกเขาไม่สามารถบังคับใครให้ใช้โค้ดนั้นได้

  • เครือข่ายบิตคอยน์ถูกตรวจสอบโดย Full Nodes นับหมื่นๆ โหนดทั่วโลก ที่ถูกรันโดยคอมพิวเตอร์ส่วนตัวของ ผู้ใช้งานธรรมดา
  • Full Node เหล่านี้มีหน้าที่เป็น "ผู้คุมกฎ" มันจะคอยตรวจสอบทุกบล็อกอย่างเข้มงวด หากเจอบล็อกใดที่ให้กำเนิดเหรียญเกินโควตาของยุคนั้น โหนดจะ ปฏิเสธบล็อกนั้นทิ้งทันที โดยไม่สนว่านักขุดจะลงแรงใช้ไฟฟ้าไปมหาศาลแค่ไหน
  • ใครที่พยายามฝืนกฎ จะถูกตัดขาดออกจากเครือข่ายหลัก และกลายเป็นการ Fork แยกสายเชนออกไปเล่นกันเอง (Hard Fork)

ความขาดแคลนสัมบูรณ์ (Absolute Scarcity)

ความมั่นใจสูงสุดของผู้ใช้งานไม่ได้เกิดจากการเชื่อใจนักพัฒนาหรือนักขุด แต่เกิดจากสโลแกน "Don't Trust, Verify" ตราบใดที่ผู้คนยังคงรันโหนดเพื่อตรวจสอบบัญชีด้วยตัวเอง นโยบายการเงิน 21 ล้านเหรียญจะคงอยู่ตลอดไป สร้างคุณสมบัติความขาดแคลนทางดิจิทัลที่แท้จริง ซึ่งป้องกันไม่ให้ความมั่งคั่งของผู้คนถูกเจือจางด้วยการพิมพ์เงินเพิ่มแบบไร้ขีดจำกัด

รู้หมือไร่? ในความเป็นจริงไม่มีหน่วย "BTC" หรือ "Satoshi" อยู่ในโค้ด

ใน Source Code ของบิตคอยน์ ยอดเงินทั้งหมดถูกจัดเก็บเป็นตัวเลขจำนวนเต็มธรรมดา (Integer) ที่เรียกว่า CAmount คำว่า BTC หรือ Satoshi เป็นเพียง "หน่วยสมมติ" หรือชื่อเรียก ที่มนุษย์และโปรแกรมกระเป๋าเงิน (Wallet) กำหนดขึ้นมาแสดงผลให้เราอ่านง่ายขึ้นเท่านั้น โดยอ้างอิงให้ 1 BTC มีค่าเท่ากับตัวเลข 100,000,000 ในระบบ

ทำไมในโลกจริง จะมีบิตคอยน์ไม่ถึง 21 ล้านเหรียญ?

ถึงแม้สมการทางคณิตศาสตร์จะกำหนดเพดานไว้ที่เกือบ 21 ล้านเหรียญ แต่ในทางปฏิบัติ จำนวนเหรียญหมุนเวียนจริง (Circulating Supply) จะไม่มีทางไปถึงจำนวนนั้นได้ตลอดกาล เนื่องจากเหตุผลเหล่านี้:

  • เหรียญสูญหาย (Lost Coins): ผู้ใช้งานทำ Private Key หาย, ลืมรหัสผ่าน, ส่งเหรียญไปผิดที่อยู่, หรือเสียชีวิตโดยไม่ได้ส่งมอบกุญแจให้ใคร เหรียญเหล่านี้จะถูกล็อกตายอยู่ในระบบและไม่สามารถนำออกมาใช้ได้อีก
  • การทำลายเหรียญโดยตั้งใจ (Unspendable Outputs): มีการใช้คำสั่งพิเศษอย่าง OP_RETURN ในการฝังข้อมูลลงบล็อกเชน ซึ่งทำให้ผลลัพธ์นั้นไม่สามารถใช้จ่ายได้อีก รวมถึง Genesis Block (50 เหรียญแรกของโลก) ที่ซาโตชิเขียนโค้ดมาในลักษณะที่มันไม่สามารถถูกนำไปใช้จ่ายได้
  • นักขุดเคลมรางวัลไม่ครบ (Unclaimed Rewards): ในอดีตมีหลายบล็อกที่นักขุดใส่ตัวเลขการรับรางวัล (Subsidy) "น้อยกว่า" สิทธิ์ที่ตัวเองควรจะได้ (เช่น Block 124724) ซึ่งระบบถือว่าบล็อกนั้นถูกต้อง แต่เหรียญส่วนต่างที่นักขุดลืมเคลมนั้น จะหายสาบสูญไปจากการผลิตอย่างถาวร
