🛡️ Node Validator

Node Validator คืออะไร มีหน้าที่อะไร?

Node Validator (ผู้ตรวจสอบ) คือโหนดที่ทำหน้าที่ ผู้คุมกฎ (Enforcer) ของเครือข่าย เมื่อมีบล็อกใหม่ถูกขุดพบ Node Validator จะรับข้อมูลมาเพื่อตรวจสอบความถูกต้องทั้งหมด (เช่น ลายเซ็นดิจิทัล, การจ่ายซ้ำซ้อน, ขนาดบล็อก, รางวัลที่เคลม) หากเจอบล็อกที่ทำผิดกติกาเพียงข้อเดียว มันจะปฏิเสธบล็อกนั้นทิ้งทันที!

ในโลกความเป็นจริง แยกจาก Node Miner ยังไง?

ในยุคแรก โหนดขุดและโหนดตรวจสอบคือเครื่องเดียวกัน แต่ปัจจุบัน แยกขาดจากกันชัดเจน โหนดบนเครือข่ายบิตคอยน์ส่วนใหญ่ (>99%) คือ Node Validator ที่ไม่มีกำลังขุดเลย! พวกเขาคือคอมพิวเตอร์ส่วนตัวของผู้ใช้งานธรรมดาทั่วโลกที่รันซอฟต์แวร์เพื่อ ปกป้องเครือข่าย (Don't Trust, Verify) ทำให้นักขุดที่มีเครื่องมหาศาลก็ไม่สามารถโกงกฎของระบบได้

Node Validator ในปัจจุบันมีกี่แบบ?

  • Archival Full Node: โหนดแบบเต็มรูปแบบที่สุด มันจะเก็บประวัติการโอนเงินทุกรายการตั้งแต่ Genesis Block (บล็อกแรก) จนถึงปัจจุบัน (กินพื้นที่หลายร้อย GB) และสามารถตรวจสอบย้อนหลังได้ 100% ด้วยตัวเองโดยไม่ต้องเชื่อใจใคร
  • Pruned Full Node: โหนดเต็มรูปแบบเหมือนแบบแรก ตรวจสอบความถูกต้องได้ 100% เท่ากัน แต่เพื่อประหยัดพื้นที่จัดเก็บ มันจะลบข้อมูลบล็อกเก่าๆ ทิ้งไปหลังจากตรวจสอบเสร็จแล้ว (เก็บเฉพาะบล็อกล่าสุด)
  • Light Node (SPV): โหนดแบบเบา (เช่น แอปพลิเคชันกระเป๋าเงินบนมือถือ) ไม่ได้เก็บข้อมูลทั้งหมดและตรวจสอบเองทั้งหมดไม่ได้ มันจะต้องพึ่งพาการดึงข้อมูลจาก Full Node ของคนอื่นเพื่อเช็คเฉพาะธุรกรรมที่เกี่ยวข้องกับตัวเองเท่านั้น
