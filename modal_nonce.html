⚙️ กลยุทธ์ Nonce

Nonce คืออะไร?

Nonce (Number used once) คือตัวเลขสุ่มที่นักขุดจะทำการเปลี่ยนไปเรื่อยๆ เพื่อนำไปรวมกับข้อมูลอื่นๆ ใน Block Header แล้วทำการ Hash ใหม่ หวังฟลุ๊คให้ได้ผลลัพธ์น้อยกว่าค่า Target

การสุ่มใน Simulator vs โลกความเป็นจริง

ในเว็บ Simulator นี้ ผู้เล่นสามารถเลือก "กลยุทธ์" การสุ่มตัวเลขได้ (เช่น สุ่มจาก 0 ไล่ขึ้นไป, สุ่มจากหลังมาหน้า, หรือสุ่มแบบ Random)

แต่ในโลกความเป็นจริง ช่องเก็บ Nonce ใน Block Header ถูกออกแบบมาให้มีขนาดเพียง 32-bit ซึ่งหมายความว่ามันเก็บตัวเลขได้สูงสุดประมาณ 4.3 พันล้านค่าเท่านั้น (0 ถึง 4,294,967,295)

ทำไมกลยุทธ์จึง "ไร้ความหมาย" ในปัจจุบัน?

ด้วยเครื่องขุด ASIC ในยุคปัจจุบันที่มีกำลังขุดระดับ Terahash/s (ล้านล้านครั้งต่อวินาที) เครื่องขุดสามารถสุ่มตัวเลขทั้ง 4.3 พันล้านค่าจนหมดเกลี้ยงได้ภายในเวลา ไม่ถึง 0.00001 วินาที!

นักขุดในโลกจริงจึงไม่มีใครมานั่งวางแผนกลยุทธ์ Nonce กันแล้ว เพราะไม่ว่าจะเริ่มสุ่มจากหน้าไปหลัง หรือหลังมาหน้า เครื่องขุดก็กวาดตัวเลขทั้งหมดรวดเดียวจบในพริบตา

อ้าว... แล้วถ้ารันจบ 4.3 พันล้านรอบแล้วยังหาไม่เจอ ทำยังไงต่อล่ะ?

  • Time Rolling: นักขุดจะทำการขยับค่า Timestamp เดินหน้าไป 1 วินาที ซึ่งจะทำให้ Block Header เปลี่ยนรูปไปทั้งหมด และเปิดโอกาสให้สุ่ม Nonce ใหม่ได้อีก 4.3 พันล้านรอบ
  • ExtraNonce: หาก Time Rolling ยังไม่พอ นักขุดจะลงไปแทรกข้อความพิเศษใน Coinbase Transaction ซึ่งจะทำให้ Merkle Root เปลี่ยนแปลง และสะเทือนมาถึง Block Header ทำให้สุ่ม Nonce ได้ใหม่อีกนับไม่ถ้วน!
